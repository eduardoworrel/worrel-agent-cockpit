00 · o atrito

Todo dia o agente começa do zero.

O cold-start invisível

Você fecha o terminal e o contexto evapora. Na próxima sessão, o agente recomete os mesmos erros, refaz a mesma investigação, pergunta o mesmo segredo. Cada nova janela é uma amnésia cara — em tokens e em paciência.

  • Correções aprendidas se perdem no histórico.
  • Padrões recorrentes nunca viram processo.
  • Contexto morre ao bater o limite da janela.

A camada que persiste

worrel fica entre você e os CLIs como memória de longo prazo. Observa, destila e devolve o conhecimento já organizado — para que cada sessão nova nasça sabendo o que a anterior aprendeu.

  • Memória de projeto injetada no contexto inicial.
  • Skills versionadas executam tarefas recorrentes.
  • Handoff estruturado quando o limite se aproxima.
01 · os pilares

Engenharia, não mágica.

Sete sistemas que transformam sessões efêmeras em conhecimento durável.

Local-first & privado

Tudo roda na sua máquina. Storage em SQLite local, zero telemetria, zero nuvem. Sem chave de API própria — usa exclusivamente a assinatura que você já tem nos CLIs.

Multi-CLI por adaptadores

Camada de adaptadores com interface comum. A v1 entrega Claude Code e OpenCode completos; a arquitetura aceita novos CLIs sem tocar no núcleo.

MCP server embutido

Qualquer agente — mesmo iniciado fora do app — lista projetos, carrega skills, reporta eventos e pede segredos via MCP. "Carregue a skill X do projeto Y" simplesmente funciona.

Auto-relato + varredura

O próprio agente se reporta no momento certo. Uma varredura em lote diferida destila o que passou, detecta padrões entre sessões e deduplica sugestões redundantes.

Motor de evolução de skills v2

Skills viram entidades versionadas com linhagem tipada — Aprendizado, Correção, Variante — métricas de saúde e evolução proativa opt-in. Aprovação humana continua sendo o padrão, sempre com diff e reversão por geração.

Análise retroativa v3

"Analisar tudo" varre todo o histórico já existente dos seus CLIs, propõe projetos e destila skills e memórias do passado — com controle estrito de orçamento de quota para você nunca ser surpreendido.

Cofre de segredos

Credenciais com escopo de projeto. Modo valor (cifrado, AES-256-GCM / Keychain) ou receita (instrução, sem custódia). Auditoria de todo acesso e mascaramento por padrão.

Handoff & terminal embutido

Perto de ~80% do limite, worrel gera um resumo estruturado e abre uma sessão encadeada. E inicia sessões dos CLIs de dentro da UI, já com a memória do projeto injetada.

02 · o loop

Quatro passos. Um ciclo que se fecha.

  1. 01

    Rode com um comando

    npx worrel@latest baixa o binário da sua plataforma, sobe o cockpit em localhost e abre o navegador — sempre na última versão. Go com a UI React embutida: binário único, sem dependências em runtime.

  2. 02

    Trabalhe normalmente

    Inicie sessões pela UI ou siga no seu terminal. worrel observa em ambos os modos via camada de adaptadores e MCP.

  3. 03

    Revise as sugestões

    Auto-relato e varredura geram uma fila de sugestões com diff e evidência. Aceite, edite, rejeite ou adie — no seu ritmo.

  4. 04

    Reinjete o conhecimento

    Skills e memórias aprovadas alimentam as próximas sessões de qualquer CLI. O ciclo recomeça — agora mais inteligente.

// quickstart
# sem clonar nada — sobe na última versão e abre o navegador
npx worrel@latest
# → http://localhost:7717

# ou, para desenvolvimento, a partir do código:
git clone https://github.com/eduardoworrel/worrel-agent-cockpit
make build && ./bin/worrel
03 · o diferencial

Sua máquina é o servidor.
E o limite.

worrel foi desenhado para nunca pedir confiança que não precisa. Não há backend remoto, não há conta, não há telemetria — nem a "anônima". O que o agente aprende sobre o seu código permanece onde o seu código está.

Armazenamento 100% local em SQLite
Zero telemetria, zero analytics
Sem chave de API própria do app
Inteligência só via sua assinatura nos CLIs
Cofre cifrado com auditoria de acesso
Retenção configurável de transcripts brutos
